浅谈模块化规范

  • 规范
  • 2017-10-08

无模块化开发时代弊病：

1.全局环境变量污染；2.命名冲突；3.依赖关系不好处理；

模块化开发的好处：

1.不污染模块外的任何代码；2.方便依赖关系管理；3.易于更新，合作，分层；

无模块化开发时代弊病：

1.全局环境变量污染；2.命名冲突；3.依赖关系不好处理；

模块化开发的好处：

1.不污染模块外的任何代码；2.方便依赖关系管理；3.易于更新，合作，分层；

何为模块化？

模块：实现特定功能的一组方法。
模块化:要求在编写代码的时候，按层次，按功能，将独立的逻辑，封装成可重用的模块，对外提供直接明了的调用接口，内部实现细节完全私有，并且模块之间的内部实现在执行期间互不干扰。

模块化开发，遵循3个规范：commonjs，AMD，CMD；
commonjs规范

1.每个文件就是一个模块，有自己的作用域；
2.在每一个文件定义的变量，函数，类，都是私有的，其他文件不可见
node.js遵循的就是common规范，不适用于浏览器，只适用于服务器端

CMD规范 同步模块定义

CMD类似于commonjs，一个文件就是一个模块的方法。
主要运用在前端当中的sea.js这个框架
sea.js是一个遵循cmd规范的javascript模块加载框架，可以事先JavaScript的模块化开发及加载机制
与jquery等JavaScript框架不同，sea.js不会扩展封装语言特性。而只是实现JavaScript的模块化及模块加载
sea.js作者：玉伯

AMD规范 异步模块定义

先定义所有依赖，然后在加载完成后的回调函数中执行，所有的模块将被异步加载，模块加载不影响后面语句的执行

requireJS是什么？

RequireJS是一个Javascript 文件和模块框架，是一个JavaScript模块加载器。有利于代码的可扩展性和重用性。它非常适合在浏览器中使用，但它也可以用在其他的脚本环境中，使用requireJS加载模块化脚本将提高代码的加载的速度。
RequireJS API 存在于RequireJS载入时创建的命名空间requirejs下。其主要API主要是下面三个函数:
define– 该函数用户创建模块。每个模块拥有一个唯一的模块ID，它被用于RequireJS的运行时函数，define函数是一个全局函数，不需要使用requirejs命名空间.
require– 该函数用于读取依赖。同样它是一个全局函数，不需要使用requirejs命名空间.
config– 该函数用于配置RequireJS.
